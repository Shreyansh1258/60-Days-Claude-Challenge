# Day 6 - Resume Optimization

## Objective
Improve ATS (Applicant Tracking System) compatibility and optimize resume structure for better keyword matching and recruiter readability.

---

## PART 1 — ATS SCORE

**Previous Score:** 52/100  
**Optimized Score:** 84/100  

---

## What Changed and Why It Improved ATS Score

### 1. Removed hyperlinks and icons from contact section
ATS parsers often fail to correctly interpret icons, symbols, and embedded hyperlinks. Converting contact details into plain text ensures clean parsing and full visibility of email, phone, and profile links.

---

### 2. Expanded Professional Summary with targeted keywords
The summary was enhanced with relevant technical keywords such as:
- Machine Learning Pipelines  
- Natural Language Processing (NLP)  
- Data Structures and Algorithms (DSA)  
- Object-Oriented Programming (OOP)

These keywords improve ATS matching for software and data-related roles.

---

### 3. Restructured Skills into categorized sections
Skills were reorganized into clear categories such as:
- Machine Learning & AI  
- Programming Languages  
- Tools & Platforms  

This helps ATS systems correctly classify and tag competencies instead of treating them as unstructured text.

---

### 4. Moved technical tools from projects to skills section
Tools like:
- Scikit-learn  
- TF-IDF  
- spaCy  
- Streamlit  

were previously embedded inside project descriptions. They were added to the Skills section to improve keyword visibility regardless of parsing order.

---

### 5. Improved project bullet points with strong action verbs
Weak phrases were replaced with stronger, impact-driven verbs such as:
- Engineered  
- Implemented  
- Deployed  
- Delivered  

This improves clarity, ownership, and perceived impact for both ATS systems and recruiters.

---

### 6. Removed multi-column layout and tables
The original resume used side-by-side sections and tables, which often break ATS parsing and cause missing or merged data. The layout was converted into a single-column structure.

---

### 7. Standardized section headings
All headings were aligned with ATS-friendly formats:
- Technical Skills  
- Projects  
- Certifications  

This ensures compatibility with keyword-based parsing systems.

---

## Key Learning Outcomes

- ATS systems rely heavily on structure and keyword placement, not design.
- Simple, single-column resumes perform better than visually complex layouts.
- Skills should be duplicated strategically (both in projects and skills section) for maximum visibility.
- Strong action verbs improve both ATS scoring and recruiter perception.
- Standardized headings significantly improve parsing accuracy.
